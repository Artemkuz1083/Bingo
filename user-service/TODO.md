## Суть сервиса

Сервис отвечает за профиль пользователя, публичные пользовательские данные и операции, связанные с балансом.

## Базовые задачи

- Определить стек сервиса и структуру проекта.
- Добавить конфигурацию через переменные окружения.
- Настроить подключение к PostgreSQL.
- Описать модель `User` с полями:
  - `id`
  - `username`
  - `email`
  - `balance`
  - `created_at`
- Создать endpoint профиля: `GET /users/me`.
- Добавить защиту endpoint-ов через JWT.
- Добавить получение пользователя по `id` для других сервисов.
- Добавить логику изменения баланса для начисления выигрыша.
- Добавить опциональный endpoint обновления профиля.
- Добавить схемы запросов и ответов.
- Добавить минимальные тесты для профиля и операций с балансом.

## Заметки

- Сервис зависит от JWT, который выдает `auth-service`.
- Обновление баланса понадобится позже для сервиса проверки победителя.
- Пароли, хеширование паролей и смена пароля относятся к `auth-service`, чтобы не дублировать ответственность.
