# Game Engine Service

ЭТО МОЁ ПРИМЕРНОЕ ПРЕДСТАВЛЕНИЕ АРХИТЕКТУРЫ

Исполнитель по ТЗ: Артём К.

Суть микросервиса: генерация BINGO-шаров и хранение состояния выпавших номеров.

Нужно реализовать микросервис, который будет генерировать случайные BINGO-шары, хранить уже выпавшие номера и отдавать текущее состояние выпадения шаров.

Сервис должен получать `game_id` от `lobby-service`, использовать Redis для состояния выпадения шаров и учитывать пользователя через JWT, который выдает `auth-service`.

Основные задачи:

- генерация шаров B1-B15, I16-I30, N31-N45, G46-G60, O61-O75;
- хранение выпавших шаров в Redis;
- хранение последнего выпавшего шара;
- таймер выпадения новых шаров;
- получение текущего состояния выпадения шаров;
- остановка генерации шаров по команде от других сервисов.

Проверка победы и фиксация победителя относятся к `winner-service`. Статус комнаты и игры (`waiting`, `active`, `finished`) относится к `lobby-service`.
