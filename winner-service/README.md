# Winner Service

ЭТО МОЁ ПРИМЕРНОЕ ПРЕДСТАВЛЕНИЕ АРХИТЕКТУРЫ

Исполнитель по ТЗ: Лиза.

Суть микросервиса: асинхронная проверка заявления BINGO и фиксация победителя.

Нужно реализовать микросервис, который будет принимать запрос игрока на BINGO, отправлять задачу в очередь RabbitMQ и проверять победу в отдельном воркере.

Сервис должен получать данные карточки из `card-service` или Redis, состояние выпавших шаров из `game-engine-service`, а пользователя определять через JWT от `auth-service`.

Основные задачи:

- endpoint для заявления BINGO;
- отправка задачи на проверку в RabbitMQ;
- воркер для обработки очереди;
- сверка карточки игрока с выпавшими шарами;
- фиксация победителя;
- получение победителя игры;
- начисление баланса победителю через `user-service`.

Этот сервис отвечает за финальное решение, есть ли победа. Он использует данные `card-service` и `game-engine-service`, но не генерирует шары и не управляет статусом комнаты напрямую.
