## Суть сервиса

Сервис отвечает за регистрацию, вход пользователя в систему и выдачу JWT-токенов.

## Базовые задачи

- Определить стек сервиса и структуру проекта.
- Добавить конфигурацию через переменные окружения.
- Настроить подключение к PostgreSQL.
- Создать endpoint регистрации: `POST /auth/register`.
- Создать endpoint входа: `POST /auth/login`.
- Реализовать хеширование паролей.
- Реализовать генерацию JWT access token.
- Реализовать проверку JWT через middleware или отдельные helper-функции.
- Добавить схемы запросов и ответов.
- Добавить обработку ошибок для уже занятых `email` и `username`.
- Добавить минимальные тесты для регистрации и входа.

## Заметки

- Используется таблица `users` в PostgreSQL.
- Поля `email` и `username` должны быть уникальными.
- Остальные сервисы будут получать `user_id` из JWT.
