# Card Service

ЭТО МОЁ ПРИМЕРНОЕ ПРЕДСТАВЛЕНИЕ АРХИТЕКТУРЫ

Исполнитель по ТЗ: Вадим.

Суть микросервиса: генерация и хранение BINGO-карточек игроков.

Нужно реализовать микросервис, который будет создавать карточку 5x5 для игрока, отдавать текущую карточку и отмечать выпавшие номера.

Сервис должен получать `user_id` из JWT, а также использовать `game_id`, чтобы понимать, к какой игре относится карточка. Основное хранилище карточек по ТЗ - Redis.

Основные задачи:

- генерация BINGO-карточки 5x5;
- уникальные номера по колонкам B, I, N, G, O;
- свободная центральная клетка `FREE`;
- сохранение карточки игрока в Redis;
- получение своей карточки;
- отметка выпавшего номера;
- подготовка данных для проверки победы.
